# 项目整理审计

> 日期：2026-06-30
> 目标：把当前项目里的主线代码、近期验证脚本、历史实验产物、废弃方案区分开，后续清理时避免误删仍在用的资产和脚本。

## 1. 当前项目主线

项目现在实际存在两条路线。

### 1.1 原始主线：图片 / sprite sheet 生成

入口：

```text
python -B -m backend.cli ...
```

核心目录：

```text
backend/
config.yaml
requirements.txt
assets/library/index.json
```

职责：

```text
prompt -> provider -> raw sheet -> split frames -> postprocess -> PNG + meta.json
```

这条线仍然是项目基础能力，暂时保留。

### 1.2 当前验证主线：参考图 + 图生视频 + 自动选帧 + 像素化

入口脚本：

```text
tools/wlai_video_generate.py
tools/mirror_video.py
tools/video_contact_sheet.py
tools/postprocess_walk_videos.py
tools/pixelize_sprite.py
```

职责：

```text
reference image
 -> seedance I2V
 -> front/back/left videos
 -> mirror left to right
 -> auto-loop 12 frames per direction
 -> pixel sprite sheet
 -> preview.html
```

当前最佳产物：

```text
assets/tasks/sprites/gothic_umbrella_walk_4dir_pixel_v5_auto12/
```

这条线是目前最应该继续推进的方向，相关脚本不应删除。

### 1.3 下一步主线：ComfyUI 参考图生成

文档：

```text
docs/plans/pixel-asset-generator/ComfyUI参考图生成接入方案.md
```

预期替换点：

```text
云端 API 生成 reference image
 -> 本地 ComfyUI 生成 reference image
```

视频生成和后处理暂时不替换。

## 4. 建议保留的代码

### 4.1 核心后端

```text
backend/cli.py
backend/config.py
backend/generator/
backend/postprocess/
backend/prompt/
backend/service/
backend/storage/
backend/terrain/
backend/resource_library.py
```

理由：

- README 中仍以这些入口作为核心流程。
- `tools/imagehub_generate.py`、`tools/gpt_image_generate.py`、`tools/gemini_image_generate.py` 仍依赖 prompt builder、config 和 storage path。
- 后处理、切图、调色板、地形素材库能力仍可复用。

### 4.2 当前视频验证工具

```text
tools/wlai_video_generate.py
tools/mirror_video.py
tools/video_contact_sheet.py
tools/postprocess_walk_videos.py
tools/pixelize_sprite.py
```

理由：

- 它们构成了当前已经验证有效的图生视频到 sprite sheet 路线。
- `postprocess_walk_videos.py` 依赖 `pixelize_sprite.py` 中的抠图、描边、高光恢复等函数。
- `mirror_video.py` 用于“只生成左侧，右侧镜像”的降成本策略。
- `video_contact_sheet.py` 用于快速检查视频动作质量。

### 4.3 暂时保留的云端生图桥接脚本

```text
tools/imagehub_generate.py
tools/gpt_image_generate.py
tools/gemini_image_generate.py
```

```text
保留，但标记为 provider/debug bridge
```

理由：

- ComfyUI 参考图流程还没接入前，云端生图仍是 fallback。
- 这些脚本也是历史测试复现入口。
- 后续 ComfyUI 稳定后，可以把它们移到 `tools/legacy/` 或只保留一个统一 provider 调试入口。

## 5. 废弃 / 候选归档项

### 5.1 明确可归档或删除

```text
gpt-test.py
tools/_row2_inspect.py
tools/splice_walk_row.py
tmp/
```

判断：

- `gpt-test.py` 是硬编码 API 测试脚本，`API_KEY` 为空，输出 `ceshi3.png`，编码内容已损坏，已有 `tools/gpt_image_generate.py` 替代。
- `tools/_row2_inspect.py` 是一次性检查 cat warrior row2 颜色统计的脚本。
- `tools/splice_walk_row.py` 是一次性修补旧 cat warrior 4x4 行走图第 2 行的补丁脚本，不属于当前视频路线。
- `tmp/` 是临时中间文件。

建议：

```text
如果还想保留历史复现价值，移动到 docs/archive 或 tools/legacy。
如果只关心当前路线，直接删除。
```

### 5.2 建议迁移到 legacy 的工具

```text
tools/animation_preview.html
tools/extract_video_frames.py
tools/make_walk_pose_guide.py
```

判断：

- `animation_preview.html` 是旧的通用手动预览页；当前 `postprocess_walk_videos.py` 已生成任务内 `preview.html`。
- `extract_video_frames.py` 是手动抽帧工具；当前自动选帧由 `postprocess_walk_videos.py --selection-mode auto-loop` 负责。
- `make_walk_pose_guide.py` 属于旧的 sprite sheet pose-guided 方案；当前路线不再依赖它。

建议：

```text
保留但移入 tools/legacy/，避免误认为当前主线入口。
```

### 5.3 机器相关脚本

```text
autodl_download_models.sh
upload_to_autodl.bat
```

判断：

- 与本机/AutoDL 环境强绑定。
- `upload_to_autodl.bat` 含硬编码远程主机、端口、本地模型路径。
- 不适合作为通用项目代码提交。

建议：

```text
保留本地使用，但不纳入 Git。
如果需要共享，应重写为 docs/comfyui_autodl_setup.md，不写死远程机器信息。
```
