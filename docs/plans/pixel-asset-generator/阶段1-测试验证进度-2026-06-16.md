# 阶段 1 测试验证进度（2026-06-16）

## 当前结论

阶段 1 的本地核心闭环已经具备：

```text
prompt -> provider -> raw sheet -> split frames -> postprocess -> PNG + meta.json
```

已验证本地 mock、dry-run、真实 Gemini 单图、真实 Gemini 多资产小物品、样式锁抽取、样式锁重处理、网站生成图导入后处理。  
今天的主要阻塞不是本地后处理，而是 `bobdong.cn/v1` 中转站在图片生成 API 返回较大 JSON/base64 响应时多次出现断流。

## 代码状态

- 默认普通生成后端已切回 `gemini`：
  - `backend: gemini`
  - `model: gemini-3.1-flash-image-preview`
  - `base_url: https://bobdong.cn/v1`
  - `api_key_env: GEMINI_API_KEY`
- 已新增 `openai_image` provider，用于 `gpt-image-2`，优先读取 `.env` 的 `GPT_API_KEY`。
- 已新增 `gemini_native` provider，但在当前中转站链路下 4x4 大图仍断流。
- `rembg` 默认关闭，默认走快速背景清理；显式设置 `MEOWASEN_ENABLE_REMBG=1` 才会加载本地模型。
- `tile` 默认 processed 输出尺寸已改成 `32x32`。
- `character` 默认 processed 输出尺寸为 `128x128`。
- `prop` / `icon` 默认 processed 输出尺寸为 `64x64`。
- splitter 已改进：对假透明棋盘格和非严格等分 sheet，先识别背景和主体连通组件，再按格子归属裁切，避免串帧。
- 已新增纯 AI 调用调试脚本：
  - `tools/debug/call_gpt_image.py`
  - 用于只调用 `gpt-image-2`，保存请求和原始/partial 响应。

```text

```

- 证据文件：
  - `assets/cb79fc80-6430-4d3d-8e48-68ec515fb395/meta.json`
